    Functional Use Case

    Outlines the main functional scenarios that describe how users interact with Kya Saabziâ€™s core features.

    Use Case 1: Sign-Up

    Element Detail
    Name Sign Up
    ID UC-01
    Description User can register themself from here
    Actors User, System
    Pre-Conditions
    • Sign Up Page with Username, Email & Password fields
    Basic Flow
    • User selects Username field and enters preferred username
    • User selects Email field and enters email
    • User selects the Password field and enters password
    • User clicks on the Register button
    • System stores the username & password in the database
    Post-Conditions User has been successfully registered

    Use Case 2: Login

    Element Detail
    Name Login
    ID UC-02
    Description User can login from here
    Actors User, System
    Pre-Conditions
    • Login Page with Email & Password fields
    Basic Flow
    • User selects Email field and enters registered username
    • User selects Password field and enters their password
    • User clicks on the Login button
    • System authenticates the user and logs them in
    Post-Conditions User has been successfully logged in.

    Use Case 3: Add dish

    Element Detail
    Name Add the dish
    ID UC-03
    Description User can post what they have cooked today with an optional note
    Actors User, System
    Pre-Conditions
    • User has been identified
    • Interface to post a dish is present
    Basic Flow
    • User enters the dish cooked today (with an optional note)
    • User presses the Submit button
    • System stores the dish and log in the database
    Post-Conditions Dish cooked today is stored.

    Use Case 4: Get recommendation

    Element Detail
    Name Dish Recommendation
    ID UC-04
    Description User can get dish recommendation based on their history and other's preferences
    Actors User, System
    Pre-Conditions
    • Others must have cooked atleast one dish
    Basic Flow
    • User gets upto three recommendations with notes by community
    • User clicks on the Get recommendation button
    Post-Conditions Recommendations were successfully provided

    Use Case 5: Get dish logs

    Element Detail
    Name Cooklogs
    ID UC-05
    Description User can see the past 7 dishes they cooked
    Actors User, System
    Pre-Conditions
    • Users must have cooked atleast one dish
    Basic Flow
      User clicks on View My Cooklogs button
    Post-Conditions Logs of the dishes cooked are provided

    Element Detail
